HTML5

HTML5 is a suite of technologies that work together to present information in a platform independent way

A foundation is markup, a declaration of content inside structural elements. Usually opening and closing tags


<article>Content</article>

Style is declared separately usually in another file that references the element and describes it's visual features


article {
    style: value;
}

JavaScript can used to manipulate markup after all web objects have been loaded. In order to dynamically alter elements


function namedAction(parameter) {
    parameter.value = 'newvalue';
}

Most web sites mix and match variying amounts of three components in different ways to create complex web interfaces

Accessibility

Accessible browsers have little concern for the stylistic or interactive components of a web interface

The prime concern for web develelopers is allowing disabled users to view content and navigate it, skipping irrelevant meta-information

Skip to Content - placed at the top of the body tag, to allow navigation to the uniuqe elements of the page

Skip to Navigation - Alternative for particularly complex sites that have a complete accessible navigation in the footer

Both elements above are often styled with display: none; to remove them from the visual design

Each link must direct the screen reader, or text browser to an internal target inside the html. This target has a unique id="value" attribute that is nowhere else in the document


<a href="#start_content" class="hidden">Skip to Content</a>
<a href="#start_navigation" class="hidden">Skip to Navigation</a>

The above links should not target sementic elements, but be an ID target. That being the only named target in the DOM, screen readers may not abide by styling that would hide these links in mainstream browsers

Box Model

The combination of border, padding and margin that surrounds each block of content

This one method of ensuring appropriate whitespace boundaries are set between on-screen components in the design

  • padding: the space between the content and the border
  • border: the section that separates on container from the outside
  • margin: the space outside the container

.style_class {
	padding-left: 1%;
	padding-right: 2%;
	padding-top: 3%;
	padding-bottom: 4%;
// is the same as 
	padding: 3% 2% 4% 1%;
}

The shorhand method (last line above) is preferred, but because adding the suffix (left, right, top, bottom) is more specific it takes precedence


#style_id {
	margin-top: 0;
	margin-bottom: 0;
	margin-left: auto;
	margin-right: auto;
// is the same as 
	margin: 0 auto; 
}

The selector above centres the block object in the middle of the page, if the block's width is less than the viewport apeture


html_element {
    border: 20px groove pink;
    padding: 30px;
    border-image: url(border.png) 30px repeat;
}

There are some specialised stylings that can be applied to to the box, including putting a background on the line around the element

NOTE

If you want the height and width measure to include padding and border values, one must add box-sizing styling to the element (below):

Inlining Block Elements

It is common practice to take blocks of content and place them inline to each other

float was a css replacement for the float attribute in the img element. Core use to contol how text flows around images. It can be used in many other situations, because of universal browser compatibility

inline-block: commonly used to inline list items <li> that are usually block elements. They have in-built space a negative right margin fixes this

table: Emulates CSS layout compatable with the <table> element and all it's children.

flexbox: A modern inlining technique with the most features for consistent display for irregular content

grid: A true whole of page layout technique to contain content in the same way a graphic designer would for pages on paper

Layout Techniques

Structures of HTML and Inline techniques combine to build the layout of a page

  • static: the default state of a element; in normal page flow, with content above affecting the element's position
  • relative: Takes the element ot of page flow, but leaves a space in flow, where the element would be, good for offsetting the element from its usual position
  • absolute: Takes element out of page flow, places the element on a new layer over layout relative to the edges of the document
  • fixed: The element is out of flow on a new layer and relative to the edges of the viewport or window the user sees, it will not scroll with the document, but is static

Input Validity

CSS has pseudo selectors that give the designer scope to style certain input states

The pattern attribute is related to must value entered input elements. Its value is a Regular Expression or Regex string

A regex has three components

  • [] - Must contain one of these characters
  • () - Must contain these strings of characters
  • {} - Must contain this quantity of characters

<input type="text" size="3" pattern="([A-Z]?NSW|QLD|SA|NT|VIC|TAS|WA|ACT{2,3})">

More than half of the pseudo classess available in CSS are related to input fomrs here are a few:

  • :valid/invalid - Looks at pattern attribute and other restricitve attributes to calculate validity and change display dynamcially
  • :required - The requried attribute can be styled to display something unique about the form element. Mostly Used with ::after and content
  • :focus - Is extra styling option that can be assigned to elements that have the cursor focused on them
